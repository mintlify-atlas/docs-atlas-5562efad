## Prerequisites

Before you begin, make sure you have the following installed:

- **Python 3.7 or higher** — `python --version` should return `3.7.x` or later
- **pandas** — used for CSV loading, data manipulation, and markdown table output

Install pandas if you don't have it:

```bash
pip install pandas
```

<Note>
  The GUI application (`sassamaru.py`) also requires **Tkinter**, which is bundled with most standard Python distributions. On Debian/Ubuntu systems you may need to install it separately with `sudo apt install python3-tk`.
</Note>

---

## Get up and running

<Steps>

<Step title="Clone the repository">

```bash
git clone https://github.com/mtgr18977/sassamaru-br-25.git
cd sassamaru-br-25
```

This creates a `sassamaru-br-25/` directory containing both entry points — `previsao.py` (CLI) and `sassamaru.py` (GUI) — alongside the model logic.

</Step>

<Step title="Set up the data file">

The model reads match history from a CSV file located at:

```text
sassamaru-br-25/br-25.csv
```

<Warning>
  Both scripts expect the data file at `sassamaru-br-25/br-25.csv` relative to the directory from which you run them. If you run the scripts from inside the `sassamaru-br-25/` folder, adjust the path in `previsao.py` accordingly.
</Warning>

The CSV must contain at minimum the following columns:

| Column           | Type    | Description                        |
| ---------------- | ------- | ---------------------------------- |
| `mandante`       | string  | Home team name                     |
| `visitante`      | string  | Away team name                     |
| `gols_mandante`  | integer | Goals scored by the home team      |
| `gols_visitante` | integer | Goals scored by the away team      |

Team names are automatically normalised (stripped and lowercased) on load, so `"Internacional"` and `" internacional "` are treated as the same club.

See [Data format](/data/data-format) for the full column specification and encoding requirements.

</Step>

<Step title="Run the CLI script">

From the project root (one level above `sassamaru-br-25/`):

```bash
python previsao.py
```

The script trains the hybrid Poisson + Elo model on all historical matches in `br-25.csv`, then prints predictions for the configured fixture list directly to your terminal.

**Example output:**

```text
# Previsão de Jogos - Modelo Híbrido Poisson + Elo

| Mandante      | Visitante        | Gols Mandante | Gols Visitante | Prob Mandante (%) | Prob Empate (%) | Prob Visitante (%) | Palpite  |
|:--------------|:-----------------|---------------:|----------------:|------------------:|----------------:|-------------------:|:---------|
| Internacional | Vitoria          |          1.23  |           0.98  |              42.1 |            29.5 |               28.4 | Mandante |
| Bahia         | Atletico Mineiro |          1.10  |           1.05  |              35.0 |            33.0 |               32.0 | Empate   |
```

<Tip>
  Pipe the output to a file to keep a record of each prediction run:
  ```bash
  python previsao.py > predictions-$(date +%F).txt
  ```
</Tip>

</Step>

<Step title="Run the GUI application">

```bash
python sassamaru.py
```

The application window opens automatically and loads `br-25.csv` on startup. From here you can:

- Enter up to **10 custom matchups** using the team name input fields
- Run **10,000 or 100,000 Monte Carlo simulations** using the corresponding buttons
- Track progress through the built-in progress bar
- View **xG rankings** and **Elo rankings** for all clubs in the dataset
- Add new match results directly to `br-25.csv` with the **Adicionar partida ao CSV** button
- Export results as a timestamped `.md` file

</Step>

</Steps>

## Interpreting the output table

Each row in the output corresponds to one predicted fixture. The columns are:

| Column               | Description                                                                                          |
| -------------------- | ---------------------------------------------------------------------------------------------------- |
| **Mandante**         | Home team                                                                                            |
| **Visitante**        | Away team                                                                                            |
| **Gols Mandante**    | Expected goals for the home team, derived from Poisson attack/defence strength and Elo influence     |
| **Gols Visitante**   | Expected goals for the away team                                                                     |
| **Prob Mandante (%)** | Probability that the home team wins                                                                 |
| **Prob Empate (%)**  | Probability of a draw                                                                                |
| **Prob Visitante (%)** | Probability that the away team wins                                                               |
| **Palpite**          | The model's top prediction: `Mandante` (home win), `Empate` (draw), or `Visitante` (away win)        |

The expected-goals figures are computed using team-specific attack and defence ratings estimated from historical data. Elo ratings (initialised at **1500**, K-factor **30**, home advantage **+30 points**) are blended in with a weight of **10%** (`ELO_INFLUENCE = 0.10`) to adjust for recent form. Goal counts are capped at **8** per team (`POISSON_MAX_GOLS = 8`) when computing outcome probabilities.

<Note>
  The "Palpite" column reflects only the highest-probability outcome. When probabilities are close — particularly in draw-range fixtures — treat all three values rather than the single label as the meaningful signal.
</Note>
